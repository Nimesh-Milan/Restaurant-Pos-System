How it works

One smooth loop,
zero paper

Every action a staff member takes feeds directly into the next step — nothing is written down, nothing is lost, nothing is calculated by hand.

🪑
01

Customer Arrives

Cashier opens the visual table map, taps the correct table, and starts a new order. Table colour changes from green to red — every other terminal sees it instantly.

Table MapMulti-terminal

02

Order is Built

Items are selected by category with a single tap. Quantities, special notes ("no chilli", "extra sauce"), and modifiers are added per item. The running total updates live.

Item NotesCategory TabsItem Photos

Order Modified if Needed

Customer changes their mind? Items can be added, removed, or the entire order transferred to a different table. Every void is logged with a reason and the cashier's name.

User roles

Built for two people.
Secured by design.

Every action in the system is tied to a user account. Admin and Cashier have completely separate screens — there is no way for a cashier to accidentally (or deliberately) access owner data.

👤
Owner / Manager

Admin

  • Full access to all 7 modules
  • Add, edit & deactivate staff accounts
  • Manage the full menu, prices & photos
  • View, filter & export all sales reports
  • Manage stock levels & supplier directory
  • Apply & approve discounts on any order
  • Configure restaurant settings & tax rate
  • Review cashier performance & activity log
Logs in from any device — office, home, or phone
💁
Counter Staff

Cashier

  • View & interact with visual table map
  • Take orders, add item notes & quantities
  • Modify, void, hold & transfer orders
  • Generate bills and apply approved discounts
  • Process cash & card payments
  • Print & reprint thermal receipts
  • Cannot access reports or analytics
  • Cannot change settings or menu prices
Screen auto-locks after inactivity — PIN to resume

Technology

B4 — The professional
architecture choice

This isn't a web app on a Chromebook. It's a real desktop application with a real database — built the way professional restaurant software is built.

☕

JavaFX Desktop App

Native Windows application — fast, reliable, no browser needed. Runs offline always. Feels like real software because it is.

Frontend
🗄️

Local MySQL Database

All data lives on your restaurant PC. Zero internet dependency for daily operations — even if the connection drops, the POS keeps running.

Data Layer
☁️

VPS Cloud Sync

A background thread pushes every transaction to a cloud server. Owner sees live data from a phone or laptop — anywhere in the world.

Cloud
🖨️

ESCPOS Thermal Printing

Prints directly to your 80mm receipt printer over USB or network. No Windows drivers needed. Fast, silent, and always reliable.
